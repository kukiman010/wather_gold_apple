# Gold Apple Price Watcher

Telegram-бот для отслеживания цен на товары в интернет-магазине [Золотое Яблоко](https://goldapple.ru). Пользователь добавляет ссылку на товар и пороговую цену — бот периодически проверяет стоимость и присылает уведомление, когда цена опускается ниже заданного значения.

Сервис для покупателей, не связан с магазином.

Тексты для регистрации бота в BotFather — в [docs/BOT_PROFILE.md](docs/BOT_PROFILE.md).

## Возможности

- Добавление товара по ссылке с сайта goldapple.ru
- Запрос пороговой цены через диалог (FSM)
- Автоматическая проверка цен по расписанию (по умолчанию — раз в час)
- Push-уведомление в Telegram при снижении цены ниже порога
- Повторное уведомление, если цена стала **ещё ниже** после предыдущего оповещения
- Список отслеживаемых товаров (`/list`) с текущей и целевой ценой
- Удаление товара из отслеживания через inline-кнопку
- Поддержка разных типов цен: обычная, со скидкой, по карте лояльности

## Как это работает

```
Пользователь                    Бот                         Gold Apple
     │                           │                               │
     │  ссылка на товар          │                               │
     ├──────────────────────────►│  Playwright + API             │
     │                           ├──────────────────────────────►│
     │                           │◄──────────────────────────────┤
     │  текущая цена, имя        │         JSON с ценой          │
     │◄──────────────────────────┤                               │
     │  порог (например 2000 ₽)  │                               │
     ├──────────────────────────►│  сохранение в PostgreSQL      │
     │                           │                               │
     │         ... каждый час ...│                               │
     │                           │  проверка всех товаров        │
     │                           ├──────────────────────────────►│
     │  🔔 цена снизилась!       │                               │
     │◄──────────────────────────┤                               │
```

1. Пользователь отправляет ссылку — бот парсит товар и показывает текущую цену.
2. Пользователь вводит порог — запись сохраняется в БД.
3. Фоновый планировщик (`scheduler.py`) обходит все товары и сравнивает цены.
4. При выполнении условий бот отправляет сообщение в личный чат пользователя.

## Настройка

### 1. База данных

Создайте базу на сервере PostgreSQL:

```sql
CREATE DATABASE tg_gold_apple;
```

Примените схему (опционально — бот создаст таблицу сам при первом запуске):

```bash
psql -U postgres -d tg_gold_apple -f tools/schema.sql
```

### 2. Переменные окружения

Скопируйте пример и заполните значения:

```bash
copy .env.example .env   # Windows
cp .env.example .env     # Linux / macOS
```

Минимально необходимо указать `BOT_TOKEN` и `DB_PASSWORD`. Токен получите у [@BotFather](https://t.me/BotFather) — см. [docs/BOT_PROFILE.md](docs/BOT_PROFILE.md).

### 3. Telegram-бот

1. Создайте бота через `/newbot` в BotFather.
2. Заполните имя, описание и команды по [docs/BOT_PROFILE.md](docs/BOT_PROFILE.md).
3. Вставьте токен в `.env`.

## Логика уведомлений

Уведомление отправляется, если **оба** условия выполнены:

1. `current_price < target_price` — текущая цена ниже порога пользователя.
2. `last_notified_price is NULL` **или** `current_price < last_notified_price` — первое срабатывание либо цена упала ещё ниже после прошлого уведомления.

Если цена выросла обратно выше порога, а потом снова упала — уведомление придёт снова (если новая цена ниже `last_notified_price` или уведомлений ещё не было).

Интервал проверки задаётся переменной `CHECK_INTERVAL_MINUTES` (по умолчанию 60 минут). Между запросами к сайту — пауза 1 секунда, чтобы снизить нагрузку.

## Парсер Gold Apple

Сайт защищён антибот-системой (страница «Checking device»). Парсер использует **Playwright** (headless Chromium):

1. Открывает goldapple.ru и проходит проверку устройства.
2. Переходит на страницу товара.
3. Запрашивает внутренний API:  
   `GET /front/api/catalog/product-card/base/v2?itemId=...&cityId=...`

### Типы цен

| Тип в API | Когда используется |
|-----------|-------------------|
| `actual` | Обычная или акционная цена |
| `old` | Зачёркнутая «старая» цена |
| `loyalty` + `bestLoyalty` | Цена по карте лояльности |

ID города (`CITY_ID`) влияет на региональные цены. По умолчанию — Москва.

## База данных

### Таблица `products`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | BIGSERIAL | Первичный ключ |
| `user_id` | BIGINT | Telegram user ID |
| `url` | TEXT | Ссылка на товар |
| `item_id` | TEXT | Числовой ID товара на сайте |
| `name` | TEXT | Название (бренд + продукт) |
| `target_price` | NUMERIC(12,2) | Порог уведомления |
| `current_price` | NUMERIC(12,2) | Последняя известная цена |
| `last_notified_price` | NUMERIC(12,2) | Цена при последнем уведомлении |
| `created_at` | TIMESTAMPTZ | Дата добавления |

Ограничение: один товар (`item_id`) на пользователя — `UNIQUE (user_id, item_id)`.

### Слои доступа к данным

- `database.py` — пул соединений psycopg2
- `databaseapi.py` — синхронные методы CRUD
- `db_async.py` — обёртки через `asyncio.to_thread` для aiogram

## Ограничения и известные особенности

- **Только goldapple.ru** — ссылки на другие магазины не обрабатываются.
- **Playwright** — первый запуск может занять время из-за прохождения anti-bot проверки; нужен установленный Chromium.
- **Один экземпляр бота** — парсер держит одну сессию браузера; несколько процессов могут конфликтовать.
- **FSM в памяти** — состояние диалога (`MemoryStorage`) теряется при перезапуске; незавершённое добавление товара нужно начать заново.
- **Сторонний сервис** — не аффилирован с «Золотым Яблоком»; соблюдайте правила использования сайта.
